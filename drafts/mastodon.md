# Mastodon

**Venue:** Mastodon · a tech instance (e.g. fosstodon.org) + a music instance · **Following weeks**

### Before you post
- Mastodon toots cap around 500 chars — keep it tight; the variants below fit.
- Be conversational, not announcement-only. Reply to people who boost/reply.
- Hashtags are how discovery works here — include the relevant ones. Disclose authorship implicitly ("I built").

---

### Tech instance (e.g. fosstodon.org) — #AudioDev #MCP #OpenSource
```
I built an open-source agent that listens to a song, figures out how its synth/keyboard part
was made, and recreates it on a real hardware synth over MIDI.

Two MCP servers (audio analysis in Python, MIDI hardware control in TypeScript) + an agent
running the tool loop between them. Runs against an on-screen mock synth too, no gear needed.
```
